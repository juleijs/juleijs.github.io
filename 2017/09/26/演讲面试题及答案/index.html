面试题：

1.split()和join()的区别？

  • 1.split()
    将一个字符串分割为子字符串，将结果作为字符串数组返回，若字符串中存在多个分割符号，亦可分割。
  • 形式：
  • stringObj.split([separator,[,limit]])
  • stringObj 必选项 string对象或者文字 该对象不会被split方法修改

  • separator 可选项 字符串或者正则表达式对象
    limit 可选项 返回数组中的元素个数

  • 普通字符串

  • 正则

  • 2.join方法

    把数组中的所有元素放入一个字符串中
    形式：
    arrayObj.join(separator);
    separator 可选项 省略该参数，则使用逗号作为分割符

  • inline:

  • none:
    元素不可见，并且不为其保留相应的位置

display被设置：none,这时元素实际上就从页面中被移走，它下面所在的元素就会被自动跟上填充。

  • 二、visibility的用法

  • visibility:visible
    元素可见，默认值

  • visibility:hidden
    元素不可见，但仍然为其保留相应的空间

  • visibility:collapse
    只对table对象起作用，能移除行货列但不会影响表格的布局。如果这个值用在table以外的对象上则表现为hidden

  • visibility:inherit
    继承上级元素的visibility值

  • 三、区别：

  • display:none;
    使用该属性后，HTML元素（对象）的宽度、高度等各种属性值都将“丢失”;
  • visibility:hidden;
    使用该属性后，HTML元素（对象）仅仅是在视觉上看不见（完全透明），而它所占据的空间位置仍然存在，也即是说它仍具有高度、宽度等属性值。

    3.ajax的提交方式有哪些?它们的区别是什么？

  • 一、提交方式有get和post
  • 二、区别：

  • 1.Get方式:
    用get方式可传送简单数据，但大小一般限制在1KB下，数据追加到url中发送（http的header传送），也就是说，浏览器将各个表单字段元素及其数据按照URL参数的格式附加在请求行中的资源路径后面。另外最重要的一点是，它会被客户端的浏览器缓存起来，那么，别人就可以从浏览器的历史记录中，读取到此客户的数据，比如帐号和密码等。因此，在某些情况下，get方法会带来严重的安全性问题。

  • 2.Post方式:
    当使用POST方式时，浏览器把各表单字段元素及其数据作为HTTP消息的实体内容发送给Web服务器，而不是作为URL地址的参数进行传递，使用POST方式传递的数据量要比使用GET方式传送的数据量大的多。 总之，GET方式传送数据量小，处理效率高，安全性低，会被缓存，而POST反之。

    要设置header的Context-Type为application/x-www-form-urlencode

    4.js闭包是什么，有什么特性？

  • 1.闭包是什么?

  • 函数中 返回函数(对象)

  • 3.闭包的特性：

  • ①.封闭性：
    外界无法访问闭包内部的数据，如果在闭包内声明变量，外界是无法访问的，除非闭包主动向外界提供访问接口；
  • ②.持久性：一般的函数，调用完毕之后，系统自动注销函数，而对于闭包来说，在外部函数被调用之后，闭包结构依然保存在
    系统中，闭包中的数据依然存在，从而实现对数据的持久使用。
  • 优点：
  • ① 减少全局变量。
  • ② 减少传递函数的参数量

  • ③ 封装；

  • 缺点：

  • 使用闭包会占有内存资源，过多的使用闭包会导致内存溢出等.

5.谈谈你对前端性能优化的理解？

前端优化的目的是提升访问速度、改善用户体验。前端页面主要包括html、css、js，可以从三个方面来优化前端性能。

1、html部分

  • 1、遵循w3c关于html的书写规范
    • （1）文档的声明，XHTML1.0 过渡型，html5直接声明即可
    • （2）页面编码，charset=“utf-8”
    • （3）html标签要正确的闭合
    • （4）html标签名、属性名必须小写，属性值要加引号
    • （5）内联元素禁止包裹块级元素
  • 2、结构语义化
    • 语义化的结构可以使代码简洁清晰，便于团队的开发和维护，同时有利于搜索引擎
  • 3、减少DOM节点，加速页面渲染
    • 简洁清新的dom结构，可以加快浏览器对dom的读取，加速整个页面的渲染速度。
  • 4、alt属性和title属性
    • title属性用来为元素提供额外说明信息
  • 5、防止src属性和link的href属性为空
    • 当值为空时，浏览器很可能会把当前页面当成其属性值加载。
  • 6、避免重定向
    • 不是绝对的避免，尽量减少
  • 7、 最小化 iframe 的数量
    • 在页面中不是使用越多就越好，因为使用大量的ifram标签会稀释你该页面的关键词密度并对其产生影响，搜索引擎还有可能认为你这是作弊，因此网站优化者建议不要过多使用虽然是一个小标签

2、css部分：

  • 1、把css放在页面head标签内
    • 样式文件放在顶部，浏览器能够有针对性的对 HTML 页面从顶到下进行解析和渲染。
  • 2、避免行内样式和内联样式
    • 使用外部调用法来引入css样式文件，有利于实现结构、表现和行为的分离。方便页面的维护和样式的修改。
  • 3、避免 CSS Expressions表达式
    • 由CSS表达式所实行的性能损失是相当大的，因为每当任何事件被触发时浏览器都需重新评估每个表达式，如窗口调整大小，鼠标移动等。
  • 4、避免使用 CSS Filter（CSS滤镜）
  • 5、精简css代码
    • a.我们在命名class或id的时候，尽量采用英文缩写，这样可以减少代码量；
    • b.删除重复性的代码，提高代码的重复使用率；
    • c.减少查询层级：如.header .logo要好过.header .top .logo；
    • d.减少查询范围：如.header>li要好过.header li；
    • e.避免TAG标签与CLASS或ID并存：如a.top、button#submit
  • 6、使用link而不是@import
    • 在IE中@import指令等同于把link标记写在HTML的底部。而这与第一条是背道而驰的。
  • 7、使用CSSS prites
    通过CSSS prites把同类图片合成一张，减少图片请求

3、js部分

  • 1、脚本放在代码页的底部
    • 对于加载页面时用到的js，可以适当的放在头部，因为页面加载通常都是从上到下的，因此js放在页面底部是会出错的。但是，通常来讲，js脚本在下载的时候，浏览器干不了其它的事儿，并且没有加在出来的dom节点在js还可能会出错，阻塞页面的渲染。
  • 2、精简js代码
    • 删除重复脚本；简洁易懂的命名也会使js文件更小；考虑到代码的可复用性，既能提高开发效率，又可以减小js文件冗余代码。
  • 3、尽量少用全局变量
    • 全局变量的问题在于它在整个Javascript应用或Web页面内共享。它们生存于同一个全局命名空间内，总可能发生命名冲突。比如，一个应用程序中两个独立的部分同时定义了同名的全局变量，却有不同的目的，很容易造成脚本问题。
  • 4、避免使用Eval
    • “eval_r()是一个魔鬼”，该函数可以将任意字符串当做js代码来执行。
  • 5、避免频繁操作DOM节点
    • 缓存已经访问过的元素；“离线”更新节点, 再将它们添加到树中；避免使用 JavaScript 输出页面布局–应该是 CSS 的事儿。
  • 6、对字符串进行循环操作，
    • 譬如替换、查找，应使用正则表达式
  • 7、遵循编码约定开发可维护的编码
    • 无论是从编码约定、命名约定、命名模式都可以制定并遵循一定的规范，提高代码的可读性，易于维护性。
